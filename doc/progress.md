# iSize 进展文档

## 2026-04-07: Web 端裁剪优化与高级功能实现

### 已完成任务
1. **裁剪功能全面升级**
   - 修复了图像裁剪的偏移计算逻辑，采用了全新的焦点坐标计算方式（Focal Point system）。
   - 改进了 Web UI 中的缩放与拖拽体验，不再出现坐标溢出或抖动。
   - 新增：双击重置裁剪设置，支持通过滑块及鼠标滚动（需配合其他键或调整触控板配置）动态放大。
2. **交互 UI 重构 (Premium Style)**
   - 全面抛弃基础的排版设计，利用 CSS 重绘了界面的颜色系统（Deep Sea / Slate主题）、排版、阴影（Glassmorphism），使界面极其精致、现代。
   - 工作台布局实现了左右分栏设计（Stage 在左侧，Controls 和 History 在右侧），直观性大幅提高。
   - 生成后，展示效果采用了规整紧凑的 Grid 布局。
3. **输出格式增强（ICO 支持）**
   - 新增了 `favicon.ico` 生成支持。
   - 内部实现了一套轻量级的底层封包打包方案（将多个尺寸如 16, 32, 48 的 PNG 合成为单个符合规范的 ICO文件），成功解决了依赖包在前端环境中的编译问题（如原 icojs 只能读取无法执行 encode）。
4. **图片压缩支持**
   - 集成并启用了 `browser-image-compression`。
   - 用户可以在侧边栏调整针对输出 PNG 质量的压缩比（从原始调整到有损压缩状态），在打包导出时自动应用该设置。

### 技术变更点
- `icon-workbench.ts`：增加图像像素边界推算与压缩算法。自定义实现 ArrayBuffer 级的 `generateIco`。
- `App.tsx`：整体外层 UI 以及布局容器的精简和现代化替换。新增状态用于管理包含 ICO 及开启压缩等特性。
- `index.css`：引入大面积的 Blur Filter、CSS Variables 调色板及更好的交互伪类设计。
- 移除了部分冗余变量依赖并修正了 TypeScript 类型错误。

### 下一步计划
- 后端（NestJS）兜底生成的完善处理链路（针对特大图片或跨平台复杂的尺寸）。
- 对小程序等非常规规格继续完善并补充默认项。

## 2026-04-07 (Phase 2): 自由裁剪与确权工作流重构

### 进一步优化任务
1. **绝对自由裁剪**
   - 移除 `offsetX` 和 `offsetY` 的边界锁定（Clamp），现在图片可以被拖拽出中心画幅以外，甚至只截图部分或边角。
   - 彻底修复 `Canvas` (在 `renderVariantBlob` 中) 重绘切图与网页 UI 视口的微小像素偏差，实现百分百“所见即所得”。
2. **两步走“确认部位”工作流**
   - 为避免不断变动的剪切参数导致误生成，引入了“确认工作流”。
   - 增加 **“确认裁剪部位”** 的主要按钮并带有视觉进度引导。用户可以随时确认/重新确认裁剪区域，确认后的母版将固定用于后续的各个端多规格衍生。
   - 分离“是否压缩”和“压缩比率”。增加独立的“启用图片压缩”复选框 Toggle。只有打开时才会加载压缩处理逻辑，关闭时使用原生质量保证无损要求。

## 2026-04-07 (Phase 3): 拖拽上传与 DOM-Canvas 坐标系对齐

### 补丁与体验修复
1. **统一 DOM 与 Canvas 的渲染原点（解决坐标偏移）**
   - 彻底解决了由于外部包裹层 (`.crop-stage`) 尺寸变化导致的 `HTML DOM` 预览与内部画布重绘脱节的问题。
   - 引入了一个严格约束为 320x320 的相对定位遮罩层 (`.crop-viewport`)，确保拖拽偏移的 CSS 坐标变换绝对等价于 `canvas.drawImage` 的参数映射。生成图片实现 100% “预览即所得”。
2. **重置超出边框的防护限制**
   - 响应产品需求“至少是在图片尺寸内”，去除了“自由平移出透明区域”的功能。
   - 恢复并修复了 `getPreviewMetrics` 中的严格 Clamp 坐标轴逻辑截断计算。现在即使用户疯狂拖动，图片主体依然会被限制在裁剪区域之内，绝对不会看到留白和透底。
3. **导出包扁平化结构 (Android 平移)**
   - 修复了 `contracts` 文件内的预设导出项，去除了嵌套的 Android 二级目录（如 `mipmap-hdpi/ic_launcher.png`）。
   - 将各个像素配置直接扁平化输出到了压缩包根目录，并且文件命名中携带了具体像素标记（如 `ic_launcher_72x72.png`）。
4. **提升无缝上手体感（拖放上传）**
   - 除了原有的“点击上传”按钮，给整个 Stage 工作台添加了文件系统的 `Drag & Drop` 原生接口支持监听。
   - 现在可以直接将要裁剪的主图拖入浏览器中处理，并且附有 Hover 提醒的视觉状态。
