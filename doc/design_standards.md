# 各端应用图标（Icon）设计基准与规格文档

本项目的核心目标是输出符合各个独立系统和平台原生审美预期的应用图标。为此，我们将部分设计规范硬编码在处理流程中（如特定圆角、底色注入等）。下表列举了引擎对各个端的图形自动化切设标准：

## 1. Web & PWA (浏览器平台)

- **favicon (16x16 / 32x32)**
  - **边缘**：直角方形 (`Square`)。
  - **背景**：保留原生透明通道。
  - **备注**：浏览器 Tab 栏通常需要透明且清晰的直角结构。

- **apple-touch-icon (180x180)**
  - **边缘**：超椭圆圆角 (`Squircle`)，圆角率约为 `22.37%`。
  - **背景**：自动填充 `纯白色 (#FFFFFF)` 防止透明度发黑（iOS 系统强规禁止 Web Clip 带透明度）。
  - **备注**：用于用户将其从 Safari 钉到手机桌面时的图标，强制应用苹果系 UI 的经典小圆角和白底。

## 2. iOS & macOS (苹果生态)

- **iOS / App Store 资产包**
  - **边缘**：超椭圆圆角 (`Squircle`)，渲染圆弧完美贴合官方平滑曲线（`border-radius` = `width * 0.2237`）。
  - **背景**：所有的 iOS 图标在打包时不允许透明通道。透明区域将自动由系统填充为 `纯白色 (#FFFFFF)`。
  - **提示**：正规的 Xcode 上传时，原图不应带圆角，由苹果审核服务器进行裁切。但为了满足内部展示、分发网页或部分套壳 App 等常见视觉化处理诉求，本项目生成 iOS 包时会**显性附加圆角渲染**。

- **macOS**
  - **边缘**：超椭圆圆角 (`Squircle`)。
  - **背景**：保留透明度，以适应不同的毛玻璃 Dock 栏与访达背景。

## 3. Android (安卓开发生态)

- **Launcher / Store Icons**
  - **边缘**：旧版标准圆角矩形 (`Rounded Rect`)，圆角率通常默认为 `8%`，或是正圆形（随着 Android 8+ Adaptive Icons 规则的普及，本工具导出的是包含透明背景的不加遮罩原图形，允许客户自配底图）。
  - **背景**：透明通道。
  - **备注**：不强加圆角和底色，因为 Android 大部分启动器自带遮罩规范（如三星、MIUI各自圆角率不同）。

## 4. Windows (微软生态)

- **UWP / Tiles / Store**
  - **边缘**：完全正方形 (`Square`)。
  - **背景**：透明。保留最广泛的正方形资产，以备开始菜单贴片调色。

---

> *附录：渲染算法实现*  
> Squircle 裁切并非标准的二次贝塞尔圆角，在我们的 Canvas 渲染引擎 (`lib/icon-workbench.ts`) 中，采用高精度的分段 `arcTo` 连接拟合出接近 Apple 工业设计的平滑超椭圆截面。如果检测到资产标记了特定的图形边界（如 shape: "squircle"），则在 `drawImage` 画布渲染之前就会执行 `clip()` 全局遮罩裁切。
